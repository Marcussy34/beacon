# Task 3 Fix-Up Report ‚Äî M4 Beacon Focus Helper: Never-Throw Contract Fixes

## What changed

### Defect 1 ‚Äî `extension/src/pid-tty.ts`: `resolvePidTty` could reject on synchronous `execFile` throw
- Wrapped the `execFile(...)` call inside the Promise executor in a `try/catch`.
- On a synchronous spawn failure (EPERM/ENOENT), the catch calls `resolve(null)` instead of letting the error propagate as a rejection.
- All existing behavior preserved (err callback ‚Üí null, empty/'??' stdout ‚Üí null, else normalizeTty).

### Defect 2a ‚Äî `extension/src/focus-terminal.ts`: `focusTerminalByTty` could reject when `show()` throws
- Wrapped the `terminal.show(); return true;` call in a `try/catch`.
- On a throw (e.g. disposed terminal): catch returns `false` ‚Äî matched but couldn't reveal ‚Üí false, never throws.
- No vscode import added; pure framework-free module stays clean.

### Defect 2b ‚Äî `extension/src/extension.ts`: fire-and-forget had no `.catch()`
- Changed `void handleFocusUri(uri)` to `void handleFocusUri(uri).catch(() => {})`.
- handleUri still returns void synchronously; any unexpected rejection is swallowed rather than becoming an unhandled rejection.

### Minor cleanup ‚Äî `extension/package-lock.json` committed
- File already existed (generated by prior `npm --prefix extension install`). Added to the commit for reproducible builds, consistent with root repo's package-lock.json.
